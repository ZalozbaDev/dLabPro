Synopsis

Finite state transducers

Description

An instance of this class may contain one or several finite state automaton graphs. Graphs consists of a state set and a transition (edge) set. Both, states and transitions, may be qualified, i.e. "carry" additional information. The states of a graph are at least qualified with a flag indicating if the state is a final (end) state of the graph. The transitions of a graph are at least qualified with two maps: the initial state and the terminal state of the transition.

A graph is represented by three tables (instances of class data):

  • Unit description table (field ud)
  • State description table (field sd)
  • Transition description table (field td)

The follwing tables give, if not empty, symbolic names for transition intupt and output symbols:

  • Input symbol table (field is)
  • Output symbol table (field os)

Adds a sequence of idSrc.nrec transitions qualified with the (input) symbol indices contained in component nIcTis of idSrc to unit nUnit. The procedure of adding the sequence depends on the value of parameter nIcTer:

  • If nIcTer is -1 the method will re-use existing transitions and states as long as unit nUnit contains a path starting at state 0 which matches the sequence. The method will branch the automaton graph (i.e. start adding new states and transitions) at the point of the first mismatch.

Synopsis

Compiles strings into FSA/FST

Description

Compiles a list of symbol strings (contained in component nIcStr of data instance idSrc) into graphs. Each string in idSrc will be compiled into one unit in this automaton instance. The name of the units is taken from component nIcName of the source instance idSrc. If nIcName is -1, the units will be named "idSrcName_X", where "idSrcName" stands for the instance name of idSrc and "X" stands for the record index in idSrc.

The resulting graph will contain one transition for each symbol in the string. The symbol alphabet is the first symbolic component of data instance idAlp. The input symbols of the automaton's transitions will be the numeric index (record index in idAlp) of the string symbols. If the option /index is set, the transitions will additionally be qualified with the unit number as output symbol. If the representation of the string by the symbols of the alphabet is unique the resulting graph will be a chain, otherwise, unless the /greedy option is set, it may contain more than one path. If the string cannot be entirely expressed by the symbols of the alphabet the resulting graph will be empty.

Strings in idSrc may contain alternatives. They are written in the following form: "prefix(alternative1|alternative2|)suffix". An empty alternative means that the whole block can be skipped. Alternatives may contain other alternatives, but the number of opened brackets should always match the number of closed ones. Therefore no symbol should contain any of the characters "(", ")" or "|".

If idSrc contains a component named W, the graphs will contain a weight component of type LSR with that weigth once per string.

The resulting graph is either connnected or emtpy.

Example


  data idLexicon
  "prth" 32 idLexicon -addcomp
  "phon" 32 idLexicon -addcomp
  1 idLexicon -allocate
  idLexicon { Mai maI }

  data idPhonemeTable
  "symb" 4 idPhonemeTable -addcomp
  4 idPhonemeTable -allocate
  idPhonemeTable { a aI I m }

  fst itLexicon
  idLexicon 1 0 idPhonemeTable itLexicon -compile

  # The result is the following structure
  #
  #              ---> (2) ---
  #             / 0        2 \
  # (0) ---> (1)-------------->(0)
  #      3             1

  # When executing in longest-match-mode ...
  idLexicon 1 0 idPhonemeTable itLexicon /greedy -compile

  # ... this structure will be created instead
  #
  # (0) ---> (1)--->(0)
  #      3       1

Parameters

   idSrc   List of strings (and, optionally, associated names)
   nIcStr   Component index of strings in idSrc
   nIcName   Component index of names in idSrc (may be -1)
   idAlp   Symbol alphabet

Options

   /greedy   Use longest match only (default: any match), result will be a chain or empty
   /push   Only valid together with /index: place output symbols at final transitions (default: place at start transitions)

Synopsis

Composes weighted transducers

Description

This function composes two finite state transducers. The resulting transducer containts composed weight symbols if the left operand has one. It also contains a new transition compened named "~TMS" which contains intermediate symbols of the composition (output symbol of left and input symbol of right operand). Futhermore the result will contain data components, which are all components of transition table of left and right operand except "~INI", "~TER", "~TOS", "~TIS" and weight symbols.

When one of the transducers contains epsilon transitions, these transitions are not composed with any transition of the other transducer. The data components originate form that other transducer would be blank, so they have to be initialized. Defaultly they will be initialized with zero. If you want to change this use an initializer for data components after the function ({ ... }).

   "~TOS":  0 -1  1 -1
   "~DA1":  5 10 15 20
  }
  
  itSrc2.td {
   "~TER":   1   1   1  (2)
   "~INI":   0   1   1   1
   "~TIS":  -1   0   1  -1
   "~TOS":   0   1   2   3

  }

Synopsis

Extracts partial automata containing specified states.

Description

Excerpts partial automaton graphs comprising all states and transitions reachable from anchor states listed in idStates. An automaton excerpt is a subset of the state and transition sets of itSrc including all extra information stored with states and transitions. The function preserves the topology (including cycles) and the original order of states and transitions.

The excerpt for each anchor state is stored in a separate unit. If a listed anchor state does not exist, an empty unit will be created. If an anchor state is unconnected, a unit containing only that state will be created.

If idIs non-NULL, only partial automata matching the input string contained therein will be extracted. Please note, that the result is still an excerpt as defined above. That means, that if matching the input requires passing through any cycles, the result will contain these cycles and thus does not list the paths matching on the input string. The input string is always read left-to-right, even if /backward is set. Epsilon symbols in the input string are ignored. Paths containing epsilon symbols will be correctly expanded.

Example

Consider the following acceptor:


  fst f;
  { "A" "B" "C" "D" "E" "F" "G" } ' f.is =;
  ""      f /fsa   -addunit;
  0    10 f        -addstates;
  0  0  1 f        -addtrans { 0 };
  0  1  2 f        -addtrans { 1 };
  0  2  3 f /final -addtrans { 2 };
  0  0  4 f        -addtrans { 1 };
  0  4  5 f        -addtrans { 2 };
  0  5  6 f /final -addtrans { 3 };
  0  5  7 f /final -addtrans { 4 };
  0  5  8 f /final -addtrans { 5 };
  0  5  9 f /final -addtrans { 6 };
  "~NAM" -1 f.sd -rindex;
  1 1 f.sd -mark;
  f.sd 8 f.sd /mark /force -tconvert;
  f.sd -unmark;

The excerpt from state 3 backward is:


  fst g;
  f 0 { 3 } 0 NULL -1 g /backward -excerpt;

The excerpt from state 0 forward accepting the string "BC" is:


  fst g;
  f 0 { 0 } 0 { 1 2 } ' 0 g /forward -excerpt;

The excerpt from state 5 forward with a maximal path length of 1 is:


  fst g;
  f 0 { 5 } 0 NULL 1 g /forward -excerpt;

Notes

  • The method modifies SD_FLG_USER1 in the state table of itSrc.

Parameters

   itSrc   The source automaton instance.
   nUnit   The source unit.
   idStates   The data instance containing state indices of unit nUnit in itSrc.
   nIcStates   The component in idStates containing the indices.
   idIs   The data instance containing a sequence of input symbols to match (can be NULL). Epsilon symbols contained in the sequence will be ignored.
   nParam   idIs is non-NULL: The component in idIs containing the input symbol sequence;
idIs is NULL: The maximal number of transitions to expand forward and/or backward (-1 for infinite).

Options

   /forward   traverse forward only (default is both directions).
   /backward   traverse backward only (default is both directions).

Synopsis

Creates a weighted segment acceptor for a symbol sequence from a multigram tree

Description

The method has two operation modes depending on the option /segment.

1. Sequence Mode (Option /segment not set)

Creates a chain whose transitions that accept symbols taken from idSeq and are weighted with the (interpolated) conditioned n-(multi)gram probablilies of each symbol in the sequence idSeq. See below for details of the n-multigram interpolation.

The output fst instance is a weighted acceptor with idSeq.nrec+1 states and idSeq.nrec transitions whose transition table has the following components:

No.Name Description
2 NC_TD_TISacceptor input symbol taken from idSeq
3 NC_TD_LSRconditional symbol probability
4 "hlen" maximal actual n-gram length used for probability calculation

2. Segmentation Graph Mode (Option /segment set)

Creates a graph with idSeq.nrec+1 states. The states will be connected by transitions spanning 1 through max_len states. The transitions respresent sequences of symbols S = (s1, ..., sN) of length N from the input sequence and will be weighted with negative logarithmic probabilities calculated from the n-multigram tree contained in unit nUnit of this instance: w = ∑i log P(si|si-O, ..., si-1), where O stands for the maximal n-gram order nOrder. The transducer input symbols of the destination instance are global transition indices in the n-multigram tree _this uniquely defining a symbol sequence.

The output fst instance is a weighted acceptor with idSeq.nrec+1 states and at most max_len·idSeq.nrec transitions whose transition table has the following components:

No.Name Description
2 NC_TD_TISglobal index of last transition of the segment in the n-multigram tree or -1 of that segment does not exist
3 NC_TD_LSRsegment weight (negative logarithmic probability)
4 "hlen" maximal actual n-gram length used for probability calculation
5 NC_TD_RC reference counter of segment in n-multigram tree or 0 if that segment does not exist
6 "len" segment length

Multigram Interpolation

If idWeights is not NULL the method performs an n-multigram interpolation (if idWeights is NULL the method just calculates n-gram probabilities or order max_len). The interpolated conditioned n-multigram probability of item sn in the symbol sequence S = (s1, s2, ..., sN) is the weighted mean value of the conditioned n-gram probabilies for this symbol for all history lengths 0 ≤ n ≤ max_len covered by the n-multigram model:

P(sn|sn-max_len...sn-1)  = r0 · 1/symbols (Zerogram)
+ r1 · h(sn) (Unigram)
+ r2 · h(sn|sn-1) (Bigram)
+ r3 · h(sn|sn-2, sn-1) (Trigram)
+ ...
+ rmax_len · h(sn|sn-max_len, ..., sn-1) (max_len-gram)

where symbols is total number of different symbols si and R = (r0, r1, ..., rmax_len) is the n-gram interpolation vector idWeights.

If the sum over the vector elements is not equal to one, the vector will be normalized. If the actual available history of a symbol is shorter than max_len due to its position in idSeq the interpolation is carried out with the longest possible history. The actually used components of the interpolation vector idWeights will be normalized to sum up to 1 in this case. Unseen events in the n-multigram tree are assumed to have the floor probability exp(-wceil).

Parameters

Synopsis

(Re-)Estimates transition probabilities

Description

(Re-)estimatea transition probabilities. If there is no probability component in the transition table one will be added. If there is another weight component the method will converted the weigths to probabilities, perform the normal operation as described for probabilities and, finally, convert the re-estimated probabilities back to the original semiring. If there is a reference counter component in the transition table the transition probabilities will be estimated using the Jeffrey smoothing algorithm (see below) from the reference counters. If there are no reference counters but non-zero probabilities the method will normalize the sum of the probabilities of all transitions leaving each state to 1. If there are only zero-probabilities the method will generate an equal probability distribution of the transitions leaving each state.

Jeffrey Smooothing of Transition Probabilities

The Jeffrey smoothing increments all reference counters of transitions leaving a particular state by rcfloor and assigns every "unseen" input symbol the reference counter rcfloor. An input symbol is "unseen" at a state if there is no transition qualified with this symbol leaving that state. The probability of a transition en leaving state v is calculated as follows:

P(en|v) =   cn + rcfloor
∑ci + symbols·rcfloor

where ci stands for the reference counter of transition vi leaving state v.

Note: The Jeffrey smoothing is only applied if the field rcfloor is positive. It only works correctly if the field symbols is properly set to the number of symbols in the acceptor input alphabet and if the automaton is deterministic. The latter is tacitly assumed but will not be checked!

MAP - Maximum a posteriori approximation

If enabled by /usemap this approximation minimizes or maximizes the entropy of transition probabilities. It is controlled by mapexp: 0 - setting MAP off, 1 - setting MAP to minimize the entropy, -1 - setting MAP to maximize the entropy. The new probability of a transition en leaving state v is calculated as follows:

P(en|v) =   cn + mapexp P(en|v) ln P(en|v)
∑ (ci + mapexp P(ei|v) ln P(ei|v))

where ci stands for the reference counter of transition vi leaving state v.

Synopsis

Calculates reference counters from transition probabilities

Description

Reconstructs (fictive) state and transition reference counters from the transition probabilities.

The state reference counters are multiples of the stationary state probabilities which may be calculated from the transposed ergodic incidence matrix of the automaton (i.e. for the calculation all final states are connected to the zero state) The reconstruction algorithm seeks for the eigenvectors of the eigenvalue 1 of the transposed incidence matrix (as the incidence matrix is stochastic, one of it eigenvalues is always one). Let P be the transposed incidence matrix and E be the unit matrix. Then the stationary state probabilities X can be calculated as (P-E)·X=0 with the constraint ∑xi=1. To implement this contraint, the algorithm adds a line 1 to the incidence matrix. To keep the incidence matrix quadratic, the algorithm then adds a column (1,1,1,...,1,0)T which adds a dummy variable to the equation system. The constant vector of this modified equation system is 1. The equation system is solved by inverting the modified incidence matrix P': X=P'-1·1

Synopsis

Synchroneous dynamic programming

Description

Determines the best path (greatest probability or smallest weight) using time (=search step) dependent transition weights and the standard (time invariant) transition weights. The two weights are aggregated by the multiplication operation of the source transducer's weight semiring (i.e. they are assumed to be "independent" of each other).

Pruning (option /prune)

The dynamic programming algorithm determines the shortest (or most probable) path to every state at every time (=search step). This guarantees finding the optimal path from start to end. Pruning means to abort search paths with high weight (low probability). This implementation prunes all paths with

Wt > Wt,max - prn_const · (Wt,max-Wt,min)

or

Pt < Pt,max - prn_const · (Pt,max-Pt,min)

respectively. Wt denotes a particular path's weight at time (=search step) t, Wt,max and Wt,min denote the maximal and minimal path weights at this time.

Pruning may result in aborting all useful paths. To make this unlikely, the implementation

  • keeps at least FST_SDP_KEEPA search paths and at least FST_SDP_KEEPR·XS paths (XS: number of states in source unit), and
  • does not prune any paths at least FST_SDP_DMZ search steps after the start and FST_SDP_DMZ search steps before the end of the search.

Note: According to Schukat-Talamazzini Pt < prn_const · Pt,max is the "generally accepted" pruning criterion. Check this out!

The search result is a chain with exactly idWeights.nrec non-epsilon input symbols. Heading non-numeric components will be ignored, missing weights are assumed to be the multiplicative neutral element of the source transducers weight semiring.

Synopsis

Connects units using a wiring table / bigram

Description

All units in isSrc will be collected into one single unit in isDst depending on idBigram. If transition counter and probability components are present, transition probabilities are recalculated. In dLabPro scripts newly added transitions may be initialized with the optional data initializer { ... }.

If the /index option is set the original unit id is appended for all exit transitions. Exit transitions are transitions that lead either to the zero node or to a node of another (former) unit. Transitions that are not exit transitions get the unit id '-1'.

If the /implicit option is set only one common new start and end node is created. (Otherwise there will be two seperate nodes.)

The data instance idBigram may be used to control the behavior of -wire. In the following, nodes having a transition to the zero node are refered to as exit nodes. Nodes that have a transition from the zero node are refered as entry nodes. Several cases are distinguished depending on the contents of idBigram and the existance of the transition probability component '~PSR' in the transition description (isSrc.td):

  1. idBigram is NULL: New start node is connceted with every unit's start node and every unit's end node is connected with new end state.
  2. idBigram contains N+1 records and N+1 components, where N is the number of units in isSrc. No transition probability component '~PSR' exists: In this case idBigram is interpreted as unit wiring table. If the value in idBigram[i,j] is greater zero then transitions are created from all exit nodes of unit i to all entry nodes of unit j.
  3. idBigram contains N+1 records and N+1 components, where N is the number of units in isSrc. A transition probability component '~PSR' exists: In this case idBigram is interpreted as unit (model) bigram.
  4. idBigram contains M+1 records and M+1 components, where M is the number of the nodes having a transition to the zero node in all units of isSrc. No transition probability componten '~PSR' exists: In this case idBigram is interpreted as node wiring table. If the value in idBigram[i,j] is greater zero then transitions are created from node i to node j.
  5. idBigram contains M+1 records and M+1 components, where M is the number of the nodes having a transition to the zero node in all units of isSrc. A transition probability component '~PSR' exists: In this case idBigram is interpreted as start/stop node (state) bigram.

In all other cases an error message is returned.

Notes

  • In cases (2) and (3) record/component 0 in idBigram desribe the transition from unit i to new end node resp. the transition from new start node to unit j. In cases (4) and (5) record/component 0 in idBigram desribe the transition from node i to new end node resp. the transition from new start node to node j. In cases (3) and (5) all transistion reference counters are reset if they exist.
  • Optimized for speed (needs only one run over all transitions).
  • If you use the option /index and initialize newly added transitions using { ... }, you must place a wildcard character '*' at the position of the newly added unit index (last component).

Calculates the conditional probability of the last symbol in a sequence.
