Synopsis

FST-based feature-value relation processing tools

Description

Weighted feature-value relations (wFVRs)[WW14] are a means of representing and processing the meaning of signals. This experimental class handles wFVRs through an FST approximation in the form of trees. Ultimately, wFVR processing is to be realized by weighted Petri net transducers.

Example

Consider the phrase "ein Meter zwanzig" ( "one meter twenty"). The meaning represented as a FVR string [WW14] is: 

    FVR[
      LDIM:0.61[
        val:1[
          DIG:1[
            o:1[
              (ein)1:1
      ] ] ] ][
        val:NaN[
          komma:NaN
      ] ][
        unit:0.7[
          (m)m:0.7
      ] ][
        val:0.36[
          CN2:0.36[
            t:0.36[
              (zwan)2(-zig):0.36
    ] ] ] ] ]

Synopsis

Creates a (weighted) FVR from a string representation

Description

The eBNF syntax of wFVR strings is [WW14]:

    FVR      = "FVR" state
    state    = "[" states statedef states "]"
    states   = [ [ states ] state ]
    statedef = comments label comments [ ":" weight ] comments
    comments = [ [ comments ] comment ]
    comment  = "(" <alpha> ")"
    label    = <alpha>
    weight   = <numeric> | "NaN"
where <alpha> denotes an arbitrary string without white spaces and special characters and <numeric> denotes a floating point number in the English locale.

If the FVR string contains comments, the method will store the space-separated string of comments in itFvr.ud.rtext.

Parameters

   src   The source string.
   itFvr   The target FST, must not be NULL.

Return value

void

Synopsis

Determines if the argument is an FVR

Description

An fst is a feature-value relation iff

  1. it is not empty and
  2. it is a tree and
  3. the only transition starting a the root state is labelled "FVR".

Parameters

   nU   The unit of itFvr to test.
   itFvr   The fst to test.

Return value

TRUE if the unit is a FVR, FALSE otherwise.

      Source :    fvrt_compile.c(164)

Description

Check a sequence FST symbol table. This function fails if there are input symbols containing a square bracket [ or ] and other characters. Furthermore this function removes all strings in parentheses (XX) from the input symbols, concatenates them and stores the concatenation in itSeq.ud.rtext. All transitions with empty string as input symbol afterwards are converted to epsilon transitions. If itSeq is NULL, no changes are mode to idS.

Parameters

   _this   This instance of CFvrtools.
   itSeq   The sequence FST in unit 0 (maybe NULL for no changes).
   idS   The symbol table to check (maybe itSeq->is or itSeq->os).
   nBO   Return pointer for opening bracket index
   nBC   Return pointer for closing bracket index

Return value

O_K if successful, a (negative) error code otherwise.

      Source :    fvrt_compile.c(425)

Description

Creates a sequence FST from a wFVR string. The format of wFVR strings is described with -from_string and in [WW14]. The resulting FST is normalized in the sense that every instance of '[' and ']' will become a separate transition and all comments "(...)" will be stripped away. The concatenation of all comments will be stored as the unit name in itSeq.

Parameters

   _this   This instance of CFvrtools.
   lpsSrc   The source string.
   itSeq   Filled with the resulting sequence FST, must not be NULL.

Return value

O_K if successful, a (negative) error code otherwise.

An FST string or sequence contained more than one input symbol for a FVR transition. For example, the instruction

will produce this warning for the symbol "EXTRA".
