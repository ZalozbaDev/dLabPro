Synopsis

dLabPro script container and executer

Description

Instances of the function class represent a piece of dLabPro code which can be used repeatedly. The code is stored as a sequence of tokens (see field tsq). Functions have a local name space, may have formal arguments and may return a value to their calling function.

The following example illustrates the usage of the function class:


  ## Moves components from one data table to another.
  ##
  ## @param idSrc
  ##          The source table
  ## @param nFirst
  ##          Zero-based index of first component to move
  ## @param nCount
  ##          Number of components to move
  ## @param idDst
  ##          The destination table (may be identical with idSrc)
  ## @return The number of components actually moved
  ##
  function -slice(idSrc,nFirst,nCount,idDst)   # Function header
  {                                            # Define function body
    idSrc idDst && not if 0 return; end        # Return 0 if idSrc or idDst are NULL
    idSrc nFirst nCount idDst -select;         # Move components
    idSrc idDst == not if                      # If source is not identical with destination >>
      idSrc nFirst nCount idSrc -delete;       #   Delete moved components from source
    end                                        # <<
    idDst.dim return;                          # Return number of actually moved components
  }                                            # End of function body

  ## Main program                              # ----------------------------------------------
  data x;                                      # Create a data table
  data y;                                      # Create another data table
  var  n;                                      # Create a variable

  -type long 5 2 x -array                      # Initialize data table x
  {
     11  22  33  44  55
    111 222 333 444 555
  };

  x 1 2 y -slice n =;                          # Call function and store return value in n
     "\n Moved ${n} component(s)." -echo;         # Print a message
  x -print;                                    # Print data table x
  y -print;                                    # Print data table y

Instances defined inside functions have a local scope and will be automatically destroyed upon leaving the function. Functions may be nested.

Function Definition Syntax


  function identifier([arg1[,arg2[,...]]]) { <dLabPro commands> }

A function definition consists of

  • a header comprising the function's identifier and a comma-separated list of formal arguments enclosed in braces and
  • a function body made of dLabPro commands enclosed in curly braces.

Please note that the function header must not contain any white spaces. By convention, function identifiers should begin with a hyphen (-).

Function Call Syntax


  [arg1[ arg2[ ...]]] identifier;

A function is called like a method of a class, i.e. the function call consists of a white space separated argument list followed by the function identifier. A semicolon can be placed after the function call to clear the stack.

-save and -restore for functions Home Top Fields Options Methods C/C++ Errors

Synopsis

Usage and limitations of file (de)serialization for functions and programs.

Description

Functions may be saved and restored like this:


  "filename.dlp" this [/zip] -save;

and


  "filename.dlp" this [/zip] -restore;

This is an experimental feature and there are many limitations.

What it can do

  • Save/restore a program in the main code (root function),
  • save/restore all instances in the root function, including nested functions even if they were defined in included scripts (i.e. you can save and restore the complete program),
  • continue a saved program after function -restore (this actually happens automatically).

What it cannot do

  • Save/restore a program in another than the root function (because this would require saving/restoring instance variables which is not supperted),
  • restore a program within an instruction (as the stack will not be restored),
  • restore the command line arguments of the root function (they will be replaced by the command line arguments of the current dLabPro session),
  • save/restore using /xml (due to limitations of the current XML stream class),
  • restore programs which use data instances which have componentes but no records (as this is not supported by the current DN3 stream implementation; the empty instances will deserialize but without any components).

dLabPro     Identifier :    /disarm
 
C/C++     Member variable :    BOOL m_bDisarm ;

Synopsis

Do not call classproc for next interpreted instance.

Description

Prevents the following instance identifier from causing any actions.

Example


  function f(s)
  {
    # ...
  }
  /disarm f -list tokens;  # Without /disarm f would be executed here!

Remarks

Upon interpretation of an instance's identifier dLabPro executes the so called class procedure. This feature is intended for classes whose instance indentifiers have another semantics than just be the reference for its members. Examples are:

  • Variables
    The interpretation of a variable's identifier pushes the variable's value onto the stack. Thus a variable can be used with exactly the same syntax as the value it stands for.
  • Functions
    The interpretation of a functions's identifiers executes the funcion. Thus a function call has the same syntax as a method call.

The /disarm option prevents dLabPro from invoking the following instance's class procedure which means it does nothing but pushing the instance onto the stack and activate it.

Assignment on NULL has no effect.

Description

The warning occurs if an assignment is made to the NULL instance:

  data x;
  x NULL =;        # Assignment ineffective

Another possible reason for this warning is trying to assign an instance to a variable:

  data x;
  var  y;
  NULL y -iset;    # Assign NULL to variable
  x y =;           # Faulty attempt to assign a new instance to the variable y

The assignment operator on instance variables always works like a -copy to the instance referred to by the variable. If you wish to assign a new instance to the variable you must use -iset.
