Synopsis

Add-Delta-Smoothing

Description

This smoothing algorithm adjusts the counts c by adding δ to each and normalize it by some factor:
cn*=  (cn + δ)·∑wnc(wn-N+1...wn)
∑wnc(wn-N+1...wn)+δ·|V|
Where |V| is the vocabulary size. In this implementation zero counts of bi- and trigrams are not adjusted because of using backoff (see: -smooth). In fact this smoothing algorithm performs quite poor.

See also

Synopsis

Good-Turing-Smoothing

Description

This algorithm uses the idea, that each unseen N-gram is estimated by higher order N-grams. In addition the assumption is made, that counts higher than a certain threshold k are reliable and needn't to be adjusted. This threshold is set by parameter nKGt in function -smooth.
c*= 
(c+1) Nc+1 +c (k+1)·Nk+1
NcN1
    , for 0<c≤k
1 - (k+1)·Nk+1
N1
In this equation Nc is the number of N-grams, which occure c times. (It's called the frequency of frequncies.) The counts greater than k will not be changed so c*=c. Zero counts can easily be adjusted by:
c*=  N1   , for c=0
Z
where Z represents the number of zerograms. In this implementation only unigrams with count zero will be adjusted, higher order N-grams will not be adjusted because of backoff (see -smooth).

See also

   -smooth

   /add

Synopsis

Witten-Bell-Smoothing

Description

This algorithm follows the idea, that each unseen N-gram can be estimated by the number of singletons. Singletons are N-grams which occure exactly once. The calculation is shown in the following equations:
cn*=  T(wn-N+1...wn-1)·N(wn-N+1...wn)   , for c(wn-N+1...wn)=0
Z(wn-N+1...wn-1)·(N(wn-N+1...wn)+T(wn-N+1...wn-1))
Where T denotes the number of types with the given history, N denotes the number of tokens with given history and Z denotes the number of zerograms with the given history.
Next thing shown is the equation for the discounted counts with c(wn-N+1w...n)>0:
cn*=  c(wn-N+1...wn)·N(wn-N+1...wn)   , for c(wn-N+1...wn)>0
N(wn-N+1...wn)+T(wn-N+1...wn-1)
Except of unigrams, N-grams which did not occure will not be smoothed, because of using backoff (see -smooth).

See also

   -smooth
   /add
   /gt

Synopsis

computes path weights for path length nIt

Description

Computes path weights for paths in FST according to transition weights found in idTrans. The path weights are calculated up to a path length of nIt transitions.

Parameters

   idTrans   array with transition weights, should contain a component ~Nldavg with transition weights and a component "~RC" with reference counters for transitions
   nIt   number of iterations / maximal path length
   idPath   desitination for path weights, component 0 contains transition weights from idTrans, component 1..nIt contains path weights of paths with length 1..nIt, component nIt+1 contains transition id

Adds trans.RCs and pathweight to histroy or initializes new trans.RCs from history

Description

If bReset is true then history for all units is reseted befor all other operations. If idTransRC is not NULL then trans.RCs and pathweight is added to history of unit nUnit. If idUnitTransCount is not NULL then new trans.RCs are calculated for all units from history and are saved to RC-component of transition table.

Parameters

   idTransRC   transition reference counters for path to add for all transitions of FST
   nPathWeight   path weight for path to add
   nUnit   unit index to add the new path to history
   bReset   reset the history of all units befor doing anything
   idUnitTransCount   number of transitions to remain, given for each unit

Synopsis

(Re-)Estimates transition propabilities, add fallback transitions

Description

This function applies a certain smoothing algorithm to a given language modell. There are three algorithms available:

  • Add-Delta-Smoothing, activated by option /add
  • Witten-Bell-Smoothing, activated by option /wb
  • Good-Turing-Smoothing, activated by option /gt

    • If you do not activate an algorithm, Good-Turing will be used.
    Please notice, that for Add-Delta-Smoothing you have to set the variable nDelta, usually with a value in intervall (0,1]. If nDelta ist zero, the algorithm will use 1. Please also notice to set the variable nKGt, which is the threshold to adjust counts (see /gt); something arround k=5 will be a good choice.
    To get better results, a fallback strategy named Backoff is implemented.

    Backoff

    This kind of fallback works as follows: Whenever you look for the probability of an N-gram, i.e. the trigram wn-2wn-1wn, and you have none, which means you never saw this trigram, so the count c(wn-2wn-1wn) equals 0, you take the probability of the N-1-gram, if it exists; otherwise you take the N-2-gram and so on.
    For the example of trigram you use the bigram wn-1wn. More formal for the example of trigrams it looks like this:
    PBO(wn-2w-1wn)  =   P*(wn-2wn-1wn),  if c(wn-2wn-1wn)  >  0
      =   α(wn-2wn-1)·P*(wn-1wn),  if c(wn-2wn-1wn) = 0 and c(wn-1wn)  >   0
      =   α(wn)·P*(wn),   otherwise
    In this equation α is a factor which assures that the probabilities sum up to 1. It is a function of the preceding string and is calculated as follows:
    α(wn-N+1...wn-1) =   1-∑wn:c(wn-N+1...wn)>0 P*(wn|wn-N+1...wn-1)
    1-∑wn:c(wn-N+1...wn)>0 P*(wn|wn-N+2...wn-1)

    P*(wn|wn-N+1...wn-1) is the smoothed probability:
    P*(wn|wn-N+1wn-1) =   c*(wn-N+1...n)
    ∑wnc(wn-N+1wn)
    where c* is the new count adjusted by the chosen smoothing algorithm.

    Parameters

       nUnit   The unit to process or -1 for all units. Please note that an eventual conversion of the weight type will affect all units and that in this case the weights of all units will be destroyed and reset to 1.
       nDelta   It only needs to be set, when using Add-Delta-Smoothing. Should be in (0,1]
       nKGt   Is the threshold for Good-Turing-Smoothing. Only used with option /gt
       itFst   The language model to process.
