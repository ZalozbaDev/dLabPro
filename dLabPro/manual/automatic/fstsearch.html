Synopsis

Search processor for wFSTs.

Description

This is a search processor for decoding in wFSTs (see class fst). The processor integrates different search algorithms in one interface. So you can use the algorithm you like for decoding. There are timevariant and timeinvaraiant algorithms available. The interface provides interative decoding for a subset of algorithms.

The search algorithm is set by the field algo. Fields named with the algorithm name are algorithm specific options. All other fields are general options.

For decoding there is an all-in-one method name -search. If you want to do mulitple decodings with one automaton or iterative decoding, you can load the automaton with -load followed by -isearch and -backtrack.

Additionally there is a tutorial for the search processor available.

Description

Synopsis

Start state in every frame

Description

Create a new initial state for every frame in the iterative search. The option may be set only for some calls of -isearch, for all calls or for the call to -search. The path weights will be normalized to path length. Without normalization path weights of paths with different starting frame are not comparable. Additinally the is a penalty for short paths configured by wnorm_off and wnorm_dec.

Warning: With this option the search will be no longer admissible due to weight normalization.

Synopsis

Backtrack from the current decoding state

Description

Performs backtracking from the current decoding state for the numpaths best active paths. It will generate one unit for every path. The transitions are copied from the source automaton with all components unless you used /fast. The total path weight is stored in the unit table in the componet "~GW".

When backtracking while iterative decoding with the option /final backtrack only paths ending at a final state in the previous time frame. Without this option paths ending at any state in the current time frame are returned. After running -isearch with /final in any case only paths ending at a final state in the current time frame are decoded.

Parameters

   itDst   Destination transducer

Synopsis

Load the source automaton

Description

This method loads the automaton to search through in the decoder. The automaton will be copied unless you use /fast.

If nUnit<0 and there are multiple units in itSrc a simplified on-the-fly composition is performed. The decoding start in the unit zero. The flag "~SUB" in the unit table tells the decoder weather the unit is the lowest layer. If it is not zero the input symbols are interpreted as unit indices which are inserted. Otherwise the input symbols are treated as usual. Currently this is only possible for algo="tp".

Restrictions to the Automaton
maximal number of layers (on-the-fly) 4
maximal number of units 2^31-2
maximal number of states per unit 2^31-2
maximal total number of transitions 2^31-2
maximal in-/output symbol index 2^31-2
pushdown memory size 8
maximal/minimal pushdown symbol 65535/-65535
uniq state index for multiple layers (general)
uniq state index for multiple layers (speech recognition)

Parameters

   itSrc   Source automaton
   nUnit   Zero-based unit index in itSrc
