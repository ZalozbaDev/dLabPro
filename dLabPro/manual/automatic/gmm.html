Synopsis

Multivariate multiclass Gaussian mixture models

Description

Instances of the gmm class represent a set of Gaussian mixture models and provide methods for density/distance computations, model tying/splitting and clustering.

Introduction

Figure 1 shows an overview of the data stored in one instance of the gmm class. The names printed in type writer letters refer to the fields of the gmm class which store the respective data.

Figure 1: GMM Overview

The basic data elements of the gmm class are N-dimensional Gaussian probability density functions (on the following referred to as "single" Gaussians). The method -density computes the pobability density of N dimensional vectors given the single Gaussians:

The exponent of equation (1), excluding the -1/2, is known as Mahalanobis distance and computed by the method -distance. The gmm class supportes two principle ways of defining Gaussians probability density functions:

  • one N-dimensional mean vector and one N-dimensional variance vector or
  • one N-dimensional mean vector and one NxN-dimensional covariance matrix.

Both methods -density and -distance are capable of computing probability densities or distances of Gaussian mixture models. A Gaussian mixture model is defined as:

where gm stands for the mixture weighting vector of the mixture model m. -density and -distance use a vector mapping object (see field mmap) in order to compute Gaussian mixtures. The vector mapping object is not only capable of calculating a weighted sum according to equation (2) but also of computing a great variety of other aggregation functions (e.q. minimum/maximum).

Setting up a Gaussian Mixture Model

Bevor using a gmm instance for computations you need to setup its Gaussian mixture models by providing mean and (co)variance data along with a mixture map (see methods -setup and -setup_ex). For memory and computation efficency the gmm class supports sharing of covariance matrices by several Gaussians. A covariance tying map may be passed to the model through the -setup_ex method.

Synopsis

This section describes the SSE2 optimizations available for the gmm class.

Description

There is an SSE2 optimized variant of the Mahalanobis distance computation which is approximately two times faster than the standard algorithm. The optimization is activated by the line


  DEFINE: GMM_SSE2 1

in the classes' definition file gmm.def. The optimized code may not be compilable on a particular platform (resulting in lots of errors in source file gmm_core_sse2.c). In such a case just comment out the DEFINE: mentioned above.

Synopsis

Inverse covariance matrices w/o main diagonal.

Description

Each record of this data table defines the inverse covariance matrix for one or several Gaussians. Only the upper right triangles (excluding the main diagonal!) of the matrices will be stored. icov will contain C records and N*(N-1)/2 components, where N stands for the feature space dimensionality mean.dim. icov does not contain the inverse variance vectors (i.e. the main diagonals of the inverse covariance matrices). These data are stored in ivar. The order of the inverse convariances in one record of icov is:
 
k(0,1) k(0,2) k(0,3) ... k(0,N-1) k(1,2) k(1,3) ... k(1,N-1) ... k(N-2,N-1)
 
The component index i of the inverse covariance k(n,m) can be calculated as
 
i = n·N - 2n - n(n-1)/2 + m - 1 for m>n and
i = m·N - 2m - m(m-1)/2 + n - 1 for m<n.

The covariance matrices can be shared by several Gaussians. In other words, icov may contain fewer records than the number of Gaussians K (which is defined by mean.nrec). The assignment is defined by the covariance tying map cmap.

Remarks

  • If the Gaussians use only diagonal covariance matrices this field is NULL.
  • This field is write protected and can only be modified through the -setup or -setup_ex methods.
  • You may use -extract in order to obtain the set of inverse convariance matrices in their usual representation (as data table with C NxN blocks) from the GMM.

Synopsis

Mixture map.

Description

The -distance and -density methods compute distance / probability density values for the single Gaussians. The results of these computations can be interpreted as a K dimensional vector whose components correspond with the single Gaussians. Passing such result vectors through the mapping object mmap defines mixture Gaussians by computing an M-dimensional output vector from the K-dimensional input vector. Among other aggregation operations the mapping object is capable of computing a set of M weighted sums thus realizing M Gaussian mixture models.

Remarks

  • This field is write protected and can only be modified through the -setup or -setup_ex methods.
  • gmm does not check the mapping object except that the input dimensionality mmap.xdim is tested to be equal to the number Kof single Gaussians mean.nrec. Particularly it is not verified that the mapping object makes up a valid Gaussian mixture model.

Synopsis

Computes (log.) probability densities for a set of feature vectors.

Description

This method computes the (logarithmic) probability densities of the feature vectors contained in idX for each of the (mixture) Gaussians of this instance. The output data table idDens will contain idX.nrec records and M (= number of mixture Gaussians) components. The value idDens[i,m] will contain the value p(xi|m) - or log(p(xi|m)) respectively - where i stands for the record index in idX and m stands for the mixture Gaussian index. The input vectors must have N (= dimensionality of Gaussians) numeric components. The data type of the components of the output table depends on the floating point precision associated with this gmm instance (float and double are possible).

Parameters

   idX   Set of feature vectors (input).
   idXmap   Computation map (input). This map is expected to contain idX.nrec records and M (see above) components. A non-zero value of idXmap[i,m] means that the probability density p(xi|m) (i.e. the density value idDens[i,m] will be computed, otherwise idDens[i,m] will be 0 or ±0.5·dceil respectively. May be NULL in order to compute all (log.) densities. This map may save a great deal of computation time when doing forced alignments.
   idDens   Destination table (output, may be identical with idX). After completion of the method the table will contain one record of M (log.) density values for each input record.

Options

   /log   Compute logarithmic probability densities. The computed values are limited to a minimum of -0.5·dceil.
   /neglog   Compute negative logarithmic probability densities. The computed values are limited to a maximum of 0.5·dceil.
   /sse2   Use SSE2 optimization.
   /nomix   Do not use mixture map

See also

   -setup

The output data table idDist will contain idX.nrec records and M (= number of mixture Gaussians) components. The value idDist[i,m] will contain the value dM(xi|m), where i stands for the record index in idX and m stands for the mixture Gaussian index. The input vectors must have N (= dimensionality of Gaussians) numeric components. The data type of the components of the output table depends on the floating point precision associated with this gmm instance (float and double are possible).

Parameters

   idX   Set of feature vectors (input).
   idXmap   Computation map (input). This map is expected to contain idX.nrec records and M (see above) components. A non-zero value of idXmap[i,m] means that the distance dM(xi|m) (i.e. the distance value idDist[i,m] will be computed, otherwise idDist[i,m] will dceil. May be NULL in order to compute all distances. This map may save a great deal of computation time when doing forced alignments.
   idDens   Destination table (output, may be identical with idX). After completion of the method the table will contain one record of M distance values for each input record.

There are two maps supported by the gmm class:

Synopsis

Splits single Gaussians into two.

Description

Splits the single Gaussians along their axis of greatest variance.

The splitting map idMap allows to prevent Gaussians from beeing split. Moreover the method will fill this table with a children-to-parent map which associates the indices of the single Gaussians before and after the splitting. There are the following operation modes:

idMapDescription
is NULLSplit all Gaussians and do not generate a children-to-parents map.
is emptySplit all Gaussians and generate a children-to-parents map.
contains at least one numeric componentSplit Gaussians k if idMap[k,0] is not zero, otherwise do not split Gaussian k, generate a children-to-parents map

After completion idMap will contain exactly one component of type long and as many records as there are split single Gaussians. The value k=idMap[k',0] means that the parent of the child (split) Gaussian k' is k.

Remarks

  • The method dissolves covariance tying, i.e. after the splitting each single Gaussian will possess an individual covariance matrix (see field cmap).
  • The method dissolves all mixture Gaussians, i.e. there are only single Gaussians left after the splitting (see field mmap).

Parameters

   nParam   Number of Gaussians to split, if nParam≤0 all Gaussians (according to idMap) will be split
   idMap   Defines which Gaussians are to be split, will be overwritten with a map which contains the parent Gaussian index for each split Gaussian.
Important: If you just want to get the children-to-parents map but not to prevent any Gaussians from being split make sure you pass an empty table for idMap! If the map is not empty it will be misinterpreted as splitting map! (see above).
