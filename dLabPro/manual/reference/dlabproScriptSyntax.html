RPN Syntax Home Top

The syntax of the dLabPro script language is based on Reverse Polish Notation (see Tutorial I, section "The syntax"). In short that means operands/arguments always prefix the operators/functions.

There are a couple of important syntactic structures:

Instantiation


  [<initializer>] <class> <instance>;

  data x;
  2 var n;

Creates an instance <instance> of <class>. If <class> is var, an optional constant <initializer> is allowed.

Method invocation


  [<arguments>] <instance> [<options>] <method> [<postfix_arguments>]
  
  n 0 x -fill;

with

<arguments> optional white space separated list of arguments,
<instance> identifier of an instance of a class (see Class Reference Manual),
<options> optional white space separated list of options,
<method> identifier of a method which is a member of instance,
<postfix_arguments> few methods use postfix arguments which have an individual syntax depending on the method.

The methods which are members of the individual classes and the arguments and options required or allowed for an individual method are described in the Class Reference Manual.

Operand / operator expressions


  ([<arguments>] <operator>)
  
  2 PI * n * sin

See Quick Reference - Table of Operators for a list of operators.

Field access


  <instance>.<field>[.<field>[...]]
  
  x.dim

Reads a field. Fields may also be accessed through the methods -set, -get and -see.

Example


  x x.nrec n - n y /rec -delete;

Deletes the last n records (table rows) of data instance x. x, x.nrec n - and n are arguments to the method -delete of instance y. /rec is an option to that method. A dLabPro instruction ends with a semicolon.

Formula Syntax Home Top

As the RPN syntax is uncomfortable for coding arithmetic expressions, dLabPro supports an alternative so called formula syntax which allows the usual function, infix and postfix notation. Formulas may be used anywhere in a dLabPro script. They are enclosed in colons. The example above may be written a bit more legible:


  x :x.nrec-n: n y /rec -delete;

Formulas may also include assignments:


  :n=x.nrec;

In this case a formula should (but does not have to) be terminated with a semicolon.

Limitations

Formulas cannot be used:

  • to call methods or functions,
  • to assign values to fields.
